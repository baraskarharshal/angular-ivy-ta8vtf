Key Points

  • MergeMap essentially is a combination of mergeAll and map.
  • MergeAll takes care of subscribing to the ‘inner’ Observable so that we no longer have to Subscribe two times as mergeAll merges the value of the ‘inner’ Observable into the ‘outer’ Observable.
  • switchMap cancels previous HTTP requests that are still in progress, while mergeMap lets all of them finish.
  • A good use case for switchMap would be an autocomplete input, where we should discard all but the latest results from the user’s input.
  • A good use case for mergeMap would be to use in log service where we need to log everything without missing anything.
  • FlatMap is an alias of mergeMap and behaves in the same way.
  • MergeMap merges values of all inner observables into outer observables and returns one observable. Emitting of values is based on completion of individual observable.
  • SwitchMap also works like mergeMap but emits value for the latest input only.
  • ConcatMap also works like mergeMap but emits all values in the same order their inputs arrived.
